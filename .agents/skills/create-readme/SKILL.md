## Role

As a senior expert software engineer make sure the README files are appealing, informative, and easy to read and understand.

## Content Expectations

- Project name
- Project description and purpose
- Technology stack used if applicable
- Prerequisites for running the project
- Installation instructions and running tests if applicable
- Project structure and file organization if applicable as tree view
- Key features and functionalities of the project
- References if applicable

## Task

1. Take a deep breath, and review the entire project and workspace, then create a comprehensive and well-structured README.md file for the project.
2. Use content expections as a guideline, if any other content is relevant and useful, ask before including it in the readme.
3. Do not overuse emojis, and keep the readme concise and to the point.
4. Do not include sections like "LICENSE", "CONTRIBUTING", "CHANGELOG", etc. There are dedicated files for those sections.
5. Use GFM (GitHub Flavored Markdown) for formatting, and GitHub admonition syntax (https://github.com/orgs/community/discussions/16925) where appropriate.
6. If you find a logo or icon for the project, use it in the readme's header.
